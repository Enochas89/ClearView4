# Vite + React + Supabase + Vercel Boilerplate

This boilerplate provides a starting point for building a web application using Vite, React, Supabase, and deploying it on Vercel.

## Features

- **Vite**: Fast development server and build tool.
- **React**: UI library.
- **Supabase**: Backend-as-a-Service (database, auth, storage).
- **Vercel**: Zero-config deployment and hosting.

## Setup

### 1. Supabase Project

1.  Go to [Supabase](https://supabase.com/) and create a new project.
2.  Once your project is created, navigate to **Project Settings** > **API**.
3.  You will find your **Project URL** and **Project API Keys**. You will need the `anon` `public` key.

### 2. Local Development

1.  Clone this repository and navigate into the directory.
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Create a `.env.local` file by copying the example file. This file will hold your secret keys and is ignored by Git.
    ```bash
    cp .env.example .env.local
    ```
4.  Open `.env.local` and add your Supabase credentials from Step 1.
    ```
    VITE_SUPABASE_URL="YOUR_SUPABASE_URL"
    VITE_SUPABASE_ANON_KEY="YOUR_SUPABASE_ANON_KEY"
    ```
5.  **Create a sample table in Supabase:** The example `App.jsx` component tries to fetch data from a `countries` table. You can create this table using the Supabase SQL Editor.
    - Go to **SQL Editor** > **New query** and run the following SQL:
    ```sql
    -- Create the table
    create table countries (
      id bigint generated by default as identity primary key,
      name text not null
    );

    -- Enable Row Level Security
    alter table countries enable row level security;

    -- Create a policy to allow public read access
    create policy "Public countries are viewable by everyone."
      on countries for select using (true);

    -- Insert some sample data
    insert into countries (name) values
      ('United States'),
      ('Canada'),
      ('Mexico');
    ```
6.  Start the development server:
    ```bash
    npm run dev
    ```
    Your app should be running on `http://localhost:5173`.

### 3. Deployment to Vercel

1.  Push your code to a GitHub repository.
2.  Go to Vercel and import your GitHub repository.
3.  Vercel will automatically detect the Vite framework.
4.  Before deploying, go to your project's **Settings** > **Environment Variables** and add `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` with the same values from your `.env.local` file.
5.  Deploy! Vercel will build and host your site.
